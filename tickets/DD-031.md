# DD-031

## Title

Add `browser.png` screenshot capture with file-path output.

## Method

TDD

## Problem

The `browser` skill can fetch rendered HTML and run browser-side automation, but it cannot yet capture a page screenshot as a first-class skill command. Users need a simple `dashboard browser.png <url>` flow that saves a PNG file with predictable file naming behavior.

## Acceptance

- `dashboard browser.png <url>` saves a screenshot to a random `/tmp/*.png` path when `--file` is omitted
- `dashboard browser.png <url> --file <path>` writes to the requested file path
- missing `.png` suffixes are normalized by appending `.png`
- existing `.png` suffixes are preserved without duplication
- the command prints the saved file path after the screenshot completes
- Docker tests and coverage pass
- docs and `Changes` are updated
- the skill repo has the release commit and release push
