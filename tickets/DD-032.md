# DD-032

## Title

Validate browser launch paths before passing them to Playwright.

## Method

TDD

## Problem

`browser.png` could fail on a real installed DD environment because browser discovery accepted a relative `bin/chrome` from `PATH`. That wrapper then failed at runtime and Playwright closed the session before the browser skill could do any work.

## Acceptance

- relative PATH hits such as `bin/chrome` are not accepted as browser binaries
- unusable wrapper scripts that fail a launchability check are ignored
- launch options only include `executablePath` when the candidate browser is validated
- Docker tests and coverage pass
- installed DD proof passes
- latest DD source proof passes
- the skill repo has the release commit and release push
